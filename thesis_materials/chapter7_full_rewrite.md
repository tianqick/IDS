# 第7章 系统测试

## 7.1 本章概述

在完成系统设计、算法实现和功能开发之后，需要通过系统测试验证系统在实际运行环境中的正确性、稳定性与可用性。系统测试不仅要检查各功能模块能否正常运行，还需要重点关注 `null` 值、空字符串、仅包含空格的输入、重复数据、非法参数、越权访问、无结果查询和不存在对象访问等边界场景，从而判断系统是否能够对不同输入和操作给出符合预期的响应。

需要说明的是，本章测试用例并非脱离代码实现进行泛化设计，而是依据当前系统中已经实现的功能进行编写。具体包括：登录与退出、首页统计展示、CSV 上传检测、检测结果查询与删除、历史记录检索、告警筛选与分页、模型新增/启用/删除、性能评估展示、用户新增/编辑/删除，以及在线流量监测的配置、启动、停止和测试提取等功能。对于当前系统中并未实现的时间范围检索、结果分页等功能，本章不再编写对应测试项。

### 7.2.2 测试方法

本章测试主要采用黑盒测试方法，从用户视角和系统接口响应角度出发，对各功能模块进行验证。测试重点包括以下几个方面：

1. 正常流程测试。验证用户按照正常操作流程使用功能时，系统能否正确完成业务处理。
2. 边界输入测试。验证 `null` 值、空值、仅空格、重复值、非法值等场景下系统是否能够给出正确提示。
3. 异常场景测试。验证查询不存在对象、删除受约束对象、参数缺失、状态不合法等情况时系统是否能够正确处理。
4. 权限控制测试。验证普通用户与管理员在功能访问范围上是否存在有效隔离。

## 7.4 测试结果分析

### 7.4.1 测试结果汇总

从上述测试用例执行结果来看，系统各功能模块均覆盖了正常流程测试、边界输入测试、异常情况测试和权限控制测试。测试内容既包括前端页面可见功能，也包括后端接口响应逻辑，基本覆盖了本文系统中已经实现的主要业务模块。

与一般功能测试相比，本章进一步增加了大量更具代表性的边界场景，例如用户名为 `null`、密码为 `null`、输入仅为空格、模型编号为空、模型编号不存在、状态字段为空、搜索条件为空、重复点击按钮、重复上传文件、删除不存在对象以及删除受关联约束对象等情况。通过这类测试，可以更全面地验证系统在异常输入条件下的鲁棒性。

总体上，系统在以下方面表现较为稳定：

1. 用户认证流程完整，能够处理 `null` 值、空字符串、特殊字符、密码错误和正常登录等情况。
2. 结果查询与展示功能较完善，能够支持结果搜索、详情查看、历史检索和记录删除等操作。
3. 管理类功能具备较好的权限控制能力，普通用户无法访问用户管理、模型管理、告警中心和流量监测等管理员专属模块。
4. 模型管理、用户管理和告警管理模块对 `null` 值、重复值、非法值、空值以及受约束删除等情况均设置了相应校验逻辑。
5. 在线流量监测模块具备配置、启动、停止和测试提取等基础管理能力，并能够对关键配置缺失场景作出提示。

### 7.4.2 系统存在的风险与改进方向

虽然当前测试结果表明系统功能设计较为完整，但从软件工程角度看，后续仍可从以下方面继续完善：

1. 增加更多自动化测试脚本，提高回归测试效率。
2. 对文件上传、模型推理和在线监测等耗时任务增加更细粒度的异常日志记录。
3. 在前端页面进一步增强错误提示的可读性和一致性，提升用户使用体验。
4. 增加更大规模数据条件下的性能测试和压力测试，以验证系统在高并发和大数据量场景下的稳定性。
