# 4.3 入侵检测模型设计

本系统的核心检测能力由深度学习入侵检测模型提供。根据 `v2.ipynb` 中的实验设计，模型以 CIC-IDS2017 网络流量数据为训练基础，对每条流量样本进行多分类识别。模型输入为经过清洗、编码和标准化处理后的流量统计特征，输出为正常流量或具体攻击类型。考虑到网络流量特征既包含局部统计关联，又存在一定的序列依赖关系，本文最终采用融合一维卷积神经网络、双向长短期记忆网络和注意力机制的混合模型，即 1D-CNN+BiLSTM+Attention 模型。

该模型的设计思路是：首先利用 1D-CNN 对流量特征序列中的局部组合模式进行提取，获得更高层次的局部特征表示；然后通过 BiLSTM 从正向和反向两个方向建模特征序列的上下文依赖关系；最后引入注意力机制，对不同位置的隐状态分配不同权重，使模型更加关注对分类结果具有较强区分能力的关键特征。经过上述处理后，模型通过全连接分类层输出各类别的预测结果。

## 4.3.1 模型输入设计

在模型训练之前，系统首先对原始 CIC-IDS2017 数据进行统一预处理。原始数据中包含流标识、源地址、目的地址、端口、时间戳、统计特征和标签等字段。由于流标识类字段更适合用于结果追踪和业务展示，直接作为模型输入容易引入与攻击类型无关的干扰信息，因此在模型训练阶段将其剔除，仅保留数值型流量统计特征作为模型输入。

数据清洗阶段主要包括异常值处理、缺失值填充、标签编码和特征标准化。对于原始数据中可能出现的正无穷、负无穷和空值，先将其转化为缺失值，再使用对应数值列的均值进行填充。标签字段采用编码方式转换为模型可以处理的类别编号。对于特征值，由于不同流量统计指标的量纲和数值范围差异较大，本文采用 Z-Score 标准化方法进行归一化处理，使各维特征在相近尺度下参与模型训练。

经过预处理后，模型保留 78 个有效流量统计特征。为了适配一维卷积层和循环神经网络层，输入数据被重构为三维张量形式，即每条样本表示为 78×1 的特征序列。若以 N 表示样本数量，则模型输入张量形状可表示为：

```text
N × 78 × 1
```

其中，N 表示批量样本数，78 表示特征维度，1 表示每个特征位置上的通道数。该输入形式既保留了每条流量样本的完整统计信息，也便于后续卷积层沿特征维度进行局部模式提取。

## 4.3.2 1D-CNN 特征提取层设计

一维卷积神经网络用于提取流量统计特征之间的局部关联模式。网络流量特征并非完全孤立，例如包长度统计、流持续时间、包间隔、流速率和标志位计数等特征往往共同反映某类攻击行为。若直接将原始特征输入循环网络，模型需要从较底层的数值特征中同时学习局部模式和全局依赖关系，训练难度较大。因此，本文首先使用 1D-CNN 对输入特征进行局部特征提取。

根据 `v2.ipynb` 中的模型结构，一维卷积层的输入通道数为 1，输出通道数为 64，卷积核大小为 3，步长为 1，并采用填充方式保持卷积前后序列长度的稳定。卷积操作后接批归一化层和 ReLU 激活函数，以增强训练稳定性和非线性表达能力。随后通过一维最大池化层对特征序列进行下采样，降低后续 BiLSTM 层的计算开销，同时保留局部区域内响应较强的特征。

1D-CNN 层的主要作用包括三个方面：第一，提取相邻流量统计特征之间的局部组合关系；第二，通过多个卷积通道学习不同类型的局部模式；第三，通过池化操作压缩序列长度，减少模型参数和训练计算量。经过该层处理后，原始 78×1 输入被转换为具有 64 个通道的高层局部特征序列，为后续时序建模提供更抽象的表示。

## 4.3.3 BiLSTM 时序建模层设计

在 1D-CNN 完成局部特征提取后，模型将卷积输出的特征序列送入 BiLSTM 层进行上下文建模。LSTM 通过输入门、遗忘门和输出门控制信息的保留与更新，能够缓解普通循环神经网络在长序列建模中容易出现的梯度消失问题。与单向 LSTM 相比，BiLSTM 同时从正向和反向两个方向学习序列依赖关系，可以获得更加完整的上下文表示。

本文模型中的 BiLSTM 采用两层结构，隐藏单元数为 64，并开启双向建模。因此，在每个序列位置上，正向 LSTM 和反向 LSTM 的隐藏状态会进行拼接，形成维度为 128 的隐状态表示。该结构能够同时利用特征序列前后位置的信息，使模型在识别复杂攻击模式时不仅依赖某个单独特征，而是综合考虑多个特征之间的上下文关系。

BiLSTM 层在模型中的作用主要体现在两个方面。一方面，它能够进一步挖掘经过卷积提取后的局部特征之间的全局依赖关系；另一方面，双向结构可以从不同方向补充特征语义，避免模型只依赖前向或后向单一路径导致的信息不足。对于入侵检测任务而言，不同攻击类型往往由多项统计特征共同决定，BiLSTM 能够增强模型对这类复合模式的表达能力。

## 4.3.4 注意力机制层设计

BiLSTM 会为特征序列中的每个位置输出一个隐状态，但不同位置对最终分类的贡献并不相同。若只使用最后一个时间步的隐状态进行分类，可能会丢失前面位置中的关键信息。为此，本文在 BiLSTM 后引入注意力机制，对不同位置的隐状态进行加权融合，使模型能够自动突出对分类任务更重要的特征表示。

注意力层首先将每个位置的隐状态映射为一个注意力得分，然后使用 Softmax 函数将得分归一化为注意力权重。最后，模型根据注意力权重对所有隐状态进行加权求和，得到一个综合上下文向量。该向量相比固定取最后隐状态的方式，能够更加充分地利用序列中不同位置的信息。

注意力机制的计算过程可概括为：

```text
e_t = W h_t
α_t = softmax(e_t)
c = Σ α_t h_t
```

其中，h_t 表示第 t 个位置的 BiLSTM 隐状态，e_t 表示对应的注意力得分，α_t 表示归一化后的注意力权重，c 表示加权融合后的上下文向量。通过该机制，模型能够在训练过程中自动学习不同特征位置的重要程度，从而提高多类别攻击识别的准确性和鲁棒性。

## 4.3.5 分类输出层设计

注意力层输出的上下文向量被送入全连接分类层完成最终类别判定。根据 `v2.ipynb` 中的实现，分类器由两层全连接结构组成：第一层将输入特征映射到 64 维隐藏表示，并在其后加入 Dropout 层；第二层将隐藏表示映射为类别输出。Dropout 比例设置为 0.5，用于在训练阶段随机丢弃部分神经元连接，降低模型对训练样本的过拟合风险。

分类输出层产生的是各类别对应的 logits。在训练阶段，模型使用交叉熵损失函数计算预测结果与真实标签之间的差异；在推理阶段，通过 Softmax 将 logits 转换为类别概率，并选取概率最高的类别作为最终预测结果。系统在业务展示中还会根据预测概率给出置信度，用于辅助用户判断检测结果的可信程度。

本文模型面向多分类入侵检测任务，分类目标包括正常流量和多种攻击流量。根据 CIC-IDS2017 数据集的业务标签，系统主要识别 1 类正常流量和 14 类攻击流量。模型训练脚本中输出层维度由标签编码后的类别数确定，以保证模型输出与实际训练标签保持一致。

## 4.3.6 模型训练策略设计

为保证模型训练和评估过程具有可比性，本文对数据集进行训练集、验证集和测试集三部分划分。根据 `v2.ipynb` 中的实现，首先从全量数据中划分 10% 作为测试集，再从剩余数据中划分约 10% 作为验证集，最终形成约 80% 训练集、10% 验证集和 10% 测试集的结构。划分过程中采用分层抽样策略，使不同类别在各数据子集中的分布尽可能保持一致，并设置固定随机种子以保证实验可复现。

训练过程中使用交叉熵损失函数作为优化目标，优化器采用 Adam，学习率设置为 0.001，批量大小设置为 512。模型在每个训练轮次结束后，会在验证集上计算准确率、精确率、召回率和加权 F1 值。其中，加权 F1 值综合考虑不同类别的精确率和召回率，适合用于类别分布不均衡的多分类检测任务。因此，本文以验证集加权 F1 值作为最优模型权重的选择依据，当某一轮模型取得更高的验证集 F1 值时，保存该轮模型参数作为当前最优权重。

`v2.ipynb` 中用于消融对比的训练轮数设置为 25 轮。训练完成后，模型使用验证集表现最优的权重进行测试集评估，从而避免仅使用最后一轮权重造成性能波动。该训练策略能够在一定程度上兼顾模型收敛效果和泛化能力。

## 4.3.7 候选模型对比与最终模型确定

为了验证不同结构对入侵检测效果的影响，`v2.ipynb` 中设计了多组候选模型进行对比，包括 LSTM、GRU、BiLSTM、1D-CNN 以及是否引入注意力机制的组合结构。该设计能够分别考察卷积特征提取、循环序列建模、双向结构和注意力机制对模型性能的贡献。

从模型设计角度看，单独使用 LSTM 或 GRU 能够建模序列依赖关系，但缺少对局部统计模式的显式提取；单独使用 1D-CNN 能够提取局部特征，但对全局上下文关系的刻画能力有限；引入 BiLSTM 后，模型能够同时利用前向和后向信息；进一步加入注意力机制后，模型能够自动强化关键特征位置。因此，1D-CNN+BiLSTM+Attention 在结构上兼顾了局部特征提取、上下文建模和关键特征选择三方面能力。

| 候选模型 | 设计目的 |

| BiLSTM | 验证双向时序建模对分类任务的影响 |

本节根据 `v2.ipynb` 中的模型设计与训练过程，完成了入侵检测模型的结构设计说明。本文最终采用 1D-CNN+BiLSTM+Attention 混合模型：1D-CNN 用于提取流量统计特征中的局部关联模式，BiLSTM 用于建模特征序列的双向上下文依赖，Attention 用于突出对分类更关键的特征位置，全连接分类层用于输出最终类别结果。该结构能够充分利用网络流量特征中的局部信息和全局依赖关系，为系统后续离线检测和在线流量监测提供模型支撑。
