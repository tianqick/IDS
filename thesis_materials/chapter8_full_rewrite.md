# 第8章 总结与展望

## 8.1 全文工作总结

本文围绕网络空间安全场景下的入侵检测问题，结合深度学习算法与平台化系统实现，设计并完成了一个基于 `1D-CNN + BiLSTM + Attention` 混合模型的入侵检测系统。论文从研究背景与应用意义出发，对入侵检测技术的发展现状、深度学习方法在网络流量分析中的应用特点以及当前网络攻击检测面临的主要问题进行了分析，并在此基础上开展了系统设计、算法实现、实验验证与平台开发等工作，形成了较为完整的研究与实现过程。

在系统设计方面，本文完成了入侵检测平台的总体架构设计，构建了基于 `Flask + Vue 3 + MySQL + PyTorch` 的系统实现方案，明确了前后端分离的实现思路，并对用户登录与权限控制、数据上传与检测任务、检测结果展示、历史记录、告警中心、模型管理、性能评估、用户管理以及在线流量监测等功能模块进行了设计与实现。同时，论文还完成了数据库结构设计，为业务数据、检测结果和系统日志的统一管理提供了支撑。

在算法实现方面，本文针对网络流量特征既具有局部统计特征、又存在一定上下文关联关系的特点，构建了 `1D-CNN + BiLSTM + Attention` 混合模型。其中，1D-CNN 用于提取局部模式特征，BiLSTM 用于建模双向时序依赖关系，注意力机制用于增强模型对关键特征区域的关注能力。通过这种多结构融合方式，模型能够更好地完成复杂网络流量的多类别分类任务。

在实验验证方面，本文以 CIC-IDS2017 数据集为基础，完成了数据预处理、训练集与测试集构建、模型训练以及多组消融实验分析。实验结果表明，本文所采用的 `1D-CNN + BiLSTM + Attention` 模型在全部候选结构中取得了最优表现，其验证集 F1 值达到 `0.996370`，测试集 Accuracy 达到 `0.996445`，测试集 F1-Score 达到 `0.996273`。这一结果说明本文构建的混合模型在网络入侵检测任务中具有较好的分类精度和综合性能，也从实验角度验证了卷积结构、双向时序建模和注意力机制之间的有效协同作用。

在系统实现方面，本文进一步将训练得到的最佳模型接入实际平台，完成了离线 CSV 流量检测与在线流量监测两类业务链路的实现。用户可以通过平台上传流量特征文件并提交后台检测任务，系统能够完成任务创建、模型推理、结果保存和告警生成；管理员则可以进一步完成模型切换、性能评估、告警处理、用户管理以及在线流量巡检等扩展操作。通过这一实现过程，本文不仅完成了算法实验，还实现了算法能力向工程平台的延伸。

在系统测试方面，本文围绕当前已经实现的功能模块，对正常流程、边界输入、异常场景和权限控制等内容进行了较完整的测试设计。例如，对登录模块中的 `null` 输入、空字符串与错误密码，对模型管理中的重复路径、非法路径与非 `.pth` 文件，对用户管理中的重复用户名、非法角色与删除受约束用户，以及对流量监测中的空网卡配置、重复启动与空 PCAP 名称等情况均进行了覆盖。测试结果表明，当前系统在主要业务流程、输入校验和角色权限控制方面具备较好的功能完整性和可用性。

总体来看，本文完成的不仅是一个单纯的深度学习模型实验，而是围绕“系统设计、模型构建、实验分析、平台实现和系统测试”形成了较完整的研究闭环。这说明深度学习方法在网络入侵检测任务中具有较强的应用潜力，也说明将检测模型进一步平台化、工程化具有较好的现实意义。

## 8.2 本文工作的不足

尽管本文已经完成了模型设计、系统实现和测试验证等工作，但从研究深度和工程完整性角度来看，仍然存在一些不足。首先，本文实验主要基于 CIC-IDS2017 数据集展开，该数据集在入侵检测研究中具有较高代表性，但其采集环境、攻击分布和流量结构与真实复杂网络场景之间仍存在差异，因此模型在更复杂环境下的泛化能力仍需进一步验证。

其次，当前系统对运行环境仍具有一定依赖，尤其是在线流量监测模块需要抓包驱动、网卡配置、Scapy 以及 Python cicflowmeter 等组件协同工作。在实际部署过程中，不同主机环境下的驱动安装、接口识别和依赖配置可能影响系统的可移植性与部署效率。

再次，虽然本文引入了注意力机制以增强模型对关键特征的关注能力，但对于模型内部决策过程的解释仍然不够充分。目前系统更多侧重于输出检测结果和统计指标，对于“模型为何将某条流量判定为某类攻击”这一问题，还缺乏更直观的解释能力与结果可视化支持。

此外，本文的系统测试仍以功能验证和边界输入测试为主，在高并发访问、长时间运行稳定性、大规模数据压力以及复杂异常恢复等方面，还缺少更深入的测试分析。因此，从工程应用角度看，系统在稳定性和健壮性方面仍有进一步完善空间。

## 8.3 未来工作展望

针对上述不足，后续研究和系统完善工作仍有较大的拓展空间。在数据层面，后续可以引入更多公开数据集或更贴近真实业务环境的网络流量数据，开展跨数据集训练与测试研究，以进一步验证模型在复杂场景下的泛化能力和鲁棒性。

在模型层面，可以继续探索更加高效的网络结构与轻量化部署方案，例如结合 Transformer 类结构、模型蒸馏或参数压缩方法，在保证检测精度的基础上降低模型推理延迟和资源消耗，从而提升系统在实时检测场景中的适用性。

在系统层面，当前平台已经具备离线检测和在线监测的基本能力，后续可以进一步完善流式处理机制、在线监测状态恢复机制以及异常容错机制，使系统能够更稳定地适应连续运行场景。同时，还可以继续增强自动化测试、日志追踪和运行监控能力，以提高系统整体工程质量。

在应用层面，未来还可以进一步强化模型解释性与检测结果可分析性，通过特征重要性分析、注意力权重可视化和辅助判定说明等方式，帮助管理员更好地理解检测结果的形成依据。此外，也可以在现有告警模块基础上继续扩展，将检测结果与安全策略执行、风险联动处置和自动化响应机制结合起来，逐步形成从“异常发现”到“辅助处置”的更完整安全闭环。

## 8.4 本章小结

本文围绕基于深度学习的入侵检测任务，完成了从系统设计、模型构建、实验分析到平台实现与系统测试的完整研究工作。实验结果表明，`1D-CNN + BiLSTM + Attention` 混合模型在网络流量多分类任务中具有较好的检测性能；系统实现结果表明，基于该模型构建的入侵检测平台能够较好地支持离线检测、结果展示、告警管理和在线流量监测等业务需求。

尽管当前工作在数据集覆盖范围、系统环境适配、模型解释性和工程化深度等方面仍存在一定不足，但总体上已经验证了深度学习方法在入侵检测系统中的应用价值。随着后续数据资源、模型方法和系统工程能力的进一步完善，本文所设计的系统仍具有较好的扩展潜力和应用前景。
